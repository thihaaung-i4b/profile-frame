## Directory Purposes

**Project Root:**
- Purpose: Single-file static HTML application
- Contains: HTML/CSS/JS monolith and asset references
- Key files: `index.html`

**frames/:**
- Purpose: Store source SVG files for frame overlays
- Contains: SVG graphics for frame designs
- Key files: `frame1.svg`, `frame2.svg`, `frame3.svg`, `logo.svg`
- Note: Currently these are embedded as base64 data URIs in `FRAME_CONFIG` array. Source SVG files exist but are not referenced by the application.

**.planning/codebase/:**
- Purpose: Analysis and documentation for development
- Contains: Architecture, structure, conventions, testing analysis docs
- Generated by: GSD codebase mapping tools
- Consumed by: GSD planning and execution tools

**.git/:**
- Purpose: Version control repository
- Managed by: Git
- Note: Standard Git internals

**.claude/:**
- Purpose: Claude workspace local settings
- Contains: Project-specific Claude configuration
- Note: Internal workspace configuration

**.artifacts/:**
- Purpose: Temporary build artifacts and outputs
- Note: Not committed to version control

## Key File Locations

**Entry Points:**
- `index.html`: Main and only HTML page. Page load triggers frame grid rendering, image preloading, and event listener attachment.

**Configuration:**
- `FRAME_CONFIG` (lines 381-385 of `index.html`): Array defining available frames with data URIs and labels
- CSS variables (lines 11-24 of `index.html`): Color theme, spacing, and responsive breakpoints

**Core Logic:**
- Frame selection: `pick()` function (lines 422-432)
- Photo loading: `loadPhoto()` function (lines 435-455)
- Canvas rendering: `render()` function (lines 465-499)
- Download: `download()` function (lines 502-521)
- Drag/pan: `setupDrag()` IIFE (lines 524-559)
- Zoom: `zoom()` function (lines 458-462)
- Reset: `reset()` function (lines 562-581)
- UI notifications: `toast()` function (lines 413-419)

**Testing:**
- No test files present

**Styling:**
- Inline `<style>` block (lines 10-313 of `index.html`)
- CSS Variables for theming
- Media queries for responsive design at breakpoints: 380px, 480px, 768px, landscape

## Naming Conventions

**Files:**
- Lowercase with hyphens: `index.html`, `frame1.svg`, `logo.svg`

**CSS Classes:**
- Kebab-case: `.frame-grid`, `.frame-card`, `.upload-area`, `.zoom-wrap`, `.btn-dl`, `.btn-rst`, `.preview-card`, `.drag-hint`
- Semantic naming indicating structure/function
- State classes: `.selected`, `.done`, `.show`, `.active`

**JavaScript Functions:**
- camelCase: `pick()`, `loadPhoto()`, `zoom()`, `render()`, `download()`, `reset()`, `toast()`, `getCanvasScale()`, `onStart()`, `onMove()`, `onEnd()`, `buildGrid()`
- Descriptive names indicating action/purpose

**JavaScript Variables:**
- camelCase: `selIdx`, `frameImg`, `photoImg`, `zoomPct`, `panX`, `panY`, `dragging`, `dragStartX`, `dragStartY`, `preloaded`, `FRAME_CONFIG`
- Constants uppercase: `FRAME_CONFIG`
- State variables prefixed with domain: `sel*` (selection), `pan*` (panning), `drag*` (dragging)

**DOM IDs:**
- camelCase: `frameGrid`, `photoIn`, `upArea`, `uicon`, `utitle`, `usub`, `udone`, `zval`, `canvas`, `ph`, `dragHint`, `dlBtn`, `toast`, `headerLogo`
- Semantic: `upArea` (upload area), `dlBtn` (download button), `ph` (placeholder), `zval` (zoom value)

**Data Attributes:**
- Not used

## Where to Add New Code

**New Frame:**
- Edit `FRAME_CONFIG` array (lines 381-385 of `index.html`)
- Add object with `src` (data URI or URL) and `name` (label) properties
- Frames are automatically preloaded and added to grid
- No other changes needed

**New Feature (e.g., filter, effect):**
- Add state variable near line 388 for new state
- Add handler function before `reset()` (around line 500)
- Add UI button/control in the content section (lines 326-365)
- Add event listener or onclick binding
- Call `render()` to update preview if state affects canvas output
- Update `reset()` function to clear new state

**New Responsive Breakpoint:**
- Add media query after line 313 in stylesheet
- Follow existing pattern: `@media (min-width: ???px) and (max-width: ???px)`

**Utility Functions:**
- Add near corresponding feature functions (e.g., image processing near `render()`)
- No dedicated utility folder - keep inline in `index.html`
